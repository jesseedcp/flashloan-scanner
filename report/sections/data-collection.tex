\section{Data Collection}

The project collects data from Ethereum mainnet. The data collection stage is designed to support both candidate generation and transaction verification. For each detected or candidate transaction, the dataset should preserve at least the transaction hash, block number, timestamp, protocol, borrower or receiver address, borrowed token, amount, fee, and verification status.

In the current report structure, this section serves as a placeholder for the final implementation details, including the exact block range, data source, and storage format used by the group.
