\section{Problem Definition and Scope}

This project studies how flash-loan-like transactions can be recognized and explained on Ethereum. The practical goal is not only to flag suspicious transactions, but also to reconstruct how a provider contract, a receiver contract, and the borrowed assets interact within one atomic transaction. Accordingly, the project asks three questions: how provider-side and user-side interactions unfold in practice, how such transactions can be identified from on-chain evidence, and how the detected results can be organized for later analysis and presentation.

To keep the project feasible, the scope is intentionally narrow. We focus on Ethereum mainnet and three representative protocols, namely Aave V3, Balancer V2, and Uniswap V2. The implementation is rule-based rather than model-based, because protocol-aware rules are easier to explain, verify, and revise within the scope of a course project.

This scope also defines what the project does not claim to solve. The current system is not a universal detector for every flash-loan-related protocol, not a production-grade monitoring platform, and not a chain-wide measurement study of all DeFi borrowing activity. Instead, it is a protocol-aware backend prototype that demonstrates how selected flash-loan and flash-swap patterns can be captured, verified, stored, and later presented in a reproducible manner.
